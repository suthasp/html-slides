📝
ต้องมีใน CLAUDE.md

  • Conventions — ใช้ PDO prepared statements, CSRF token ทุก form

  • Domain Rules — กฎธุรกิจเฉพาะโปรเจกต์
💡 เก็บ CLAUDE.md ต่ำกว่า 500 บรรทัด — ถ้ายาวกว่านี้ให้แตกไปไว้ใน docs/ แล้ว link มา
ตัวอย่าง CLAUDE.md
markdown # Project: MedRecord v1.0

## Tech Stack
- PHP 8.2 + MySQL 8 (PDO prepared statements)
- Tailwind CSS + Glassmorphism dark theme
- Chart.js, LINE Messaging API

## Conventions
- ทุกไฟล์ต้องมี CSRF token
- ใช้ Buddhist Era (พ.ศ.) ในการแสดงวันที่

- รัน npm test ก่อน commit เสมอ
- Coverage: 80%+ ทุก module

## Git Workflow
- Branch: feature/*, fix/*, hotfix/*
- Commit prefix: feat, fix, docs, test, chore

Extension Types
ประเภทการขยายระบบ Claude Code
🔧
⌨️
Slash Commands
Manual trigger — ผู้ใช้พิมพ์ /cmd เพื่อเรียกใช้
เขียนเป็นไฟล์ .md ใน .claude/commands/
ใช้กับงานที่ทำซ้ำๆ เช่น /review, /deploy, /test-all
🎯
Skills
Auto-trigger — Claude ตัดสินใจเองตาม context
โครงสร้างซับซ้อน: SKILL.md + scripts/ + references/ + assets/
เหมาะกับ workflow ที่ต้องการ reference หลายชิ้น
🤖
Agents (Subagents)
Parallel AI instances — Claude อีกตัวที่มี context แยก
นิยามด้วย .yml ใน .claude/agents/
ใช้สำหรับ code-review, security-audit, devops
🔌
Plugins
Deep system extension — ขยายความสามารถระดับ core
ลงทะเบียนใน settings.json → plugins[]
เหมาะกับ team tools ที่ใช้ข้ามโปรเจกต์

SECTION05
Hook Events
เหตุการณ์ที่ดักจับและตอบสนองได้
🪝
Hook Eventเรียกเมื่อใช้ทำอะไร
PreToolUseก่อน Claude เรียก toolSecret scan, permission check
PostToolUseหลัง Claude เรียก toolAuto-format, lint, log
SessionStartเปิด session ใหม่โหลด env, แจ้งทีม
SessionEndปิด sessionSave summary, cleanup
PreCommitก่อน git commitRun full test, security scan
Notificationเมื่อ tool ล้มเหลวส่ง LINE/Telegram alert
ℹ️ Hooks = shell scripts ตายตัว ไม่ใช่ AI — ทำงานเหมือน git hooks แต่ trigger ด้วย Claude events
settings.json
json {
  "hooks": {
    "PreToolUse": "./scripts/secret-scan.sh",
    "PostToolUse": "./scripts/auto-format.sh",
    "SessionStart": "./scripts/load-env.sh",
    "PreCommit": "./scripts/full-test.sh"
  },
  "deniedTools": [
    "Bash(rm -rf*)",
    "Bash(DROP TABLE*)"
  ]
}

Skill Structure

🎯
โครงสร้าง Skill
.claude/skills/code-review/
├── SKILL.md # Instructions + metadata
├── scripts/ # Shell/Python scripts
│   └── lint-check.sh
├── references/ # Docs loaded on-demand
│   └── owasp-top10.md
└── assets/ # Templates & static files
     └── review-template.md
💡 Skills vs Commands: Commands = trigger เอง / Skills = Claude ตัดสินใจใช้ตาม context
SKILL.md frontmatter
yaml ---
name: code-review
description: |
  # ใช้ description บอก Claude ว่าจะ trigger เมื่อไร
  Triggers thorough review before commit.
  Activate when user says: "review", "ตรวจ code"
allowed-tools: [Read, Bash, Grep]
---

## Process
1. รัน scripts/lint-check.sh
2. โหลด references/owasp-top10.md ถ้าเจอ auth
3. สรุปด้วย assets/review-template.md
⚠️ description ที่ดี = Skill ทำงานถูกต้อง — เขียนให้ชัดว่า trigger เมื่อไร ใช้ keyword ที่ผู้ใช้จะพิมพ์จริงๆ

Agent Team Patterns
รูปแบบการทำงานของทีม Agent
🤝
🎼
Orchestrator
Main agent ควบคุม specialists — แบ่งงานให้ subagent แต่ละตัว แล้ว merge ผลลัพธ์
main → A, B, C → merge
🔗
Pipeline
ทำงานต่อเนื่องแบบ chain — output ของ A เป็น input ของ B ตามลำดับที่กำหนด
A → B → C (sequential)
🗺️
Map-Reduce
แยกงานใหญ่เป็น chunks → ทำคู่ขนาน → รวมผล เช่น analyze 100 files พร้อมกัน
split → parallel → merge
👁️
Supervisor
Agent หนึ่งคอย monitor และตรวจสอบงานของ agents อื่น — แก้ไขถ้าผิดพลาด
supervisor → workers
🐝
Swarm
Agents เท่าๆ กันทำงานร่วมกัน ไม่มีลำดับชั้น — ดีสำหรับ exploratory tasks
A ⟷ B ⟷ C (equal)

Getting Started
เริ่มต้นใช้งาน Claude Code ใน 6 ขั้น
🚀
1
ติดตั้ง Claude Code CLI
npm i -g @anthropic-ai/claude-code
2
เข้าโปรเจกต์
cd your-project && claude
3
สร้าง CLAUDE.md อัตโนมัติ (⭐ ขั้นสำคัญที่สุด)
/init — Claude จะอ่านโปรเจกต์แล้วสร้างให้
4
สร้าง folders สำหรับ extensions
mkdir -p .claude/{commands,skills,agents}
5
ลงทะเบียน MCP servers
nano .claude/.mcp.json
6
เริ่ม build!
claude แล้ว Claude พร้อมทำงาน
ตัวอย่าง .mcp.json
json {
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@mcp/github"],
      "env": { "GITHUB_TOKEN": "${env:GH_TOKEN}" }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@mcp/postgres"],
      "env": { "DATABASE_URL": "${env:DB_URL}" }
    }
  }
}
🚀 หลัง /init ให้ปรับแต่ง CLAUDE.md ที่สร้างมา เพิ่ม domain rules เฉพาะโปรเจกต์

SECTION12
Security Best Practices
ความปลอดภัยที่ต้องรู้ก่อนใช้ Claude Code จริง
🔒
หัวข้อทำอย่างไร
Secretsไม่เก็บใน CLAUDE.md เด็ดขาด — ใช้ env vars เสมอ
.env.exampleใช้เป็น template สำหรับทีม (ไม่มีค่าจริง)
settings.local.jsonต้องอยู่ใน .gitignore เสมอ
PreCommit Hookเปิด secret scanning ก่อน commit ทุกครั้ง
MCP Scopeให้ permission ขั้นต่ำที่จำเป็น (least privilege)
Secret Rotationเปลี่ยน secrets ทุก 90 วัน — สร้าง quarterly checklist
🚨 กฎเหล็ก: ถ้า secret หลุดขึ้น git ครั้งหนึ่ง — ต้อง rotate ทันที แม้ลบ commit แล้ว เพราะมีคนเก็บ history ไว้แล้ว
Pre-commit hook สแกน secret
bash # .git/hooks/pre-commit
PATTERNS=(
  'sk-[a-zA-Z0-9]{32,}'         # OpenAI
  'AIza[0-9A-Za-z_-]{35}'      # Google
  'AKIA[0-9A-Z]{16}'          # AWS
  'ghp_[a-zA-Z0-9]{36}'       # GitHub PAT
)

for p in "${PATTERNS[@]}"; do
  if git diff --cached | grep -qE "$p"; then
    echo "🚨 Secret detected! Aborting."
    exit 1
  fi
done

CLAUDE.md Anti-Patterns
5 อย่างที่ทำให้ CLAUDE.md ใช้ไม่ได้ผล
❌
#Anti-patternปัญหาแก้ยังไง
01500+ บรรทัดContext bloat — ช้าและสับสนแตกเป็นไฟล์ใน docs/ แล้ว link
02"Write good code"กว้างเกินไป Claude ไม่รู้ทำอะไรเขียนเป็นกฎเฉพาะ + ตัวอย่าง code
03Duplicate docsแก้ทีต้องไล่หลายที่Link ไปยัง source — ห้าม copy
04ไม่มี test guidanceClaude ข้ามการเขียน testระบุ framework + coverage + pattern
05ไม่มี error patternshandle error ไม่สม่ำเสมอใส่ตัวอย่าง try-catch + log format
Bad vs Good
markdown ❌ BAD: "Write good code"
❌ BAD: "Be careful with security"
❌ BAD: "Test things properly"

✅ GOOD: ระบุเฉพาะเจาะจง
- ใช้ PDO prepared statements เสมอ
- เช็ค RBAC ทุก endpoint: $user->can('...')
- ทุก function ต้องมี test ครอบคลุม:
  - Happy path (1 case)
  - Edge cases (3+: empty, null, max)
  - Error case (1+)
- Coverage ≥ 80% (fail CI ถ้าต่ำกว่า)

❌ BAD: We use Jest.
✅ GOOD: Vitest 1.x (ไม่ใช่ Jest!)

Debugging & Logging
แก้บั๊กและตรวจการทำงานของ Claude Code
🔍
คำสั่ง / Hookใช้เมื่อ
--verboseเปิด trace log ของทุก tool call
/costดู token spend ปัจจุบัน + top consumers
/resumeเล่น session ที่ล้มเหลวกลับมาใหม่
Notification hookส่ง alert เมื่อ tool call ล้มเหลว
~/.claude/logs/ดู hook logs ย้อนหลัง
ℹ️ ถ้า Skill ไม่ trigger ให้ตรวจ description ใน SKILL.md — Claude ใช้ description ตัดสินใจว่าจะใช้ skill หรือไม่
Debug commands
bash # เปิด verbose mode — เห็น tool call ทั้งหมด
claude --verbose

# ดู token spend
/cost

# Resume session ที่ล้มเหลว
claude --resume session_abc123

# ตรวจ hook ทำงานไหม
chmod +x scripts/precommit.sh
./scripts/precommit.sh # ทดสอบรันเอง

# ดู MCP server logs
cat ~/.claude/mcp-logs/github-2026.log

# Restart + test MCP
claude --restart
npx -y @mcp/github --test

Pro Tips
เคล็ดลับระดับมือโปรจากคนใช้จริง
⭐
🐝
Subagents = Parallel Research
ให้ subagent ทำ research หลายเรื่องพร้อมกัน — เร็วกว่าทำเรียงลำดับ 3× เช่น compare 3 payment gateways พร้อมกัน
📏
CLAUDE.md ≤ 500 บรรทัด
เกินกว่านี้ context จะบวม — แตกเป็น docs/ แล้ว reference ด้วย relative path
🚫
settings.local.json → .gitignore
ห้าม commit ข้อมูลส่วนตัวขึ้น git เด็ดขาด — ตรวจ .gitignore ก่อน push ทุกครั้ง
📚
Skills > Long Prompts
คำสั่งซับซ้อนให้เขียนเป็น skill ไม่ใช่ prompt ยาวๆ ใน chat — นำกลับมาใช้ได้ทุกโปรเจกต์
⚡
Hooks = Deterministic, Skills = AI
Hooks คือ shell scripts ทำงานตายตัว / Skills คือ AI ตัดสินใจตาม context — ใช้ให้ถูกบทบาท
🪄
/init = Auto CLAUDE.md
ให้ Claude สร้าง CLAUDE.md ให้ก่อนเสมอ แล้วค่อยปรับเพิ่ม domain rules — ไม่ต้องเริ่มจากศูนย์
🎓 BUGpairoj Signature Pattern: สร้าง baseline plugin ที่ bundle skills + commands + agents มาตรฐาน — โปรเจกต์ใหม่แค่ copy 5 folders แล้วได้ stack ครบ (PDO, RBAC, CSRF, LINE, Buddhist Era, glassmorphism UI)
